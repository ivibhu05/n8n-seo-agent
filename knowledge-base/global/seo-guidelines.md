# SEO Writing Guidelines — Global

## Keyword Placement Rules

- **Primary keyword**: must appear in Title (H1), within first 100 words, at least one H2, and conclusion paragraph
- **Primary keyword density**: 1.0%–1.5% of total word count (do not exceed)
- **Secondary keywords**: distribute naturally throughout — 1 mention each minimum
- **LSI terms**: weave in contextually, not forced
- **No keyword stuffing**: if it sounds unnatural, remove it

## Title (H1) Rules

- Include primary keyword naturally
- Max 60 characters for SEO title tag
- Must be compelling — answer "what will I get from reading this?"
- No clickbait that doesn't match content
- Avoid: "Ultimate Guide to X" unless it actually is comprehensive

## Meta Description

- 140–160 characters
- Include primary keyword once, naturally
- Include a value proposition or CTA
- Must match what the content actually delivers

## Heading Structure

- One H1 per page (the title)
- H2s for main sections
- H3s for sub-points within H2 sections
- Never skip levels (no H4 without H3)
- Include keywords in at least 2–3 H2s naturally
- H2s should form a logical reading flow even if read in isolation

## Internal Linking

- 3–7 internal links per piece (adjust by word count)
- Use descriptive anchor text — never "click here" or "read more"
- Link to relevant, high-authority internal pages
- Prioritize: money pages (product, pricing, signup) for commercial content
- For informational content: link to related blog posts and topic clusters
- Do not link the same URL twice in one piece

## External Linking

- 1–3 high-authority external links per piece (gov, edu, established media)
- Never link to competitors
- Open in new tab (add note for developer: target="\_blank")

## Content Freshness

- For statistics and data: always note the year (2024, 2025)
- Avoid "recently" or "lately" — be specific with dates
- For evergreen content: write in present tense

## URL Slug (to suggest at end of each piece)

- Lowercase, hyphen-separated
- Max 5 words
- Include primary keyword
- No dates in slug unless it's a time-sensitive piece

## Schema Markup Notes

- Blog posts: Article schema
- How-to guides: HowTo schema
- Listicles: ItemList schema
- FAQs: FAQPage schema — add an FAQ section where relevant

## EEAT Signals

- Mention specific data, case studies, or examples
- Attribute quotes and statistics to credible sources
- For YMYL topics: extra care on accuracy — cite official sources
- Include author expertise signals where applicable

## Content Quality Checklist

- [ ] Answers the search intent fully
- [ ] No fluff — every paragraph adds value
- [ ] Primary keyword in first 100 words
- [ ] Meta title ≤60 chars, meta description ≤160 chars
- [ ] 3–7 internal links with descriptive anchors
- [ ] External links to authoritative sources
- [ ] Conclusion with clear next step / CTA
- [ ] Suggested URL slug included
- [ ] No duplicate content from other published pieces
